# Russian Text Simplifier

Небольшое Spring Boot-приложение для адаптации русскоязычных текстов под уровни **B1** и **B2**.

Пользователь:
- вводит исходный текст,
- выбирает шаблон промпта,
- выбирает целевой уровень сложности,
- получает адаптированный текст,
- накапливает результаты в рамках текущей сессии,
- скачивает **Excel-отчет** с проверкой качества адаптации через **Текстометр API**.

После скачивания отчета история текущей сессии очищается.

---

## Что делает приложение

### 1. Адаптация текста
Приложение отправляет исходный текст в `TranslatorService`, где:
- выбирается нужный шаблон промпта,
- подставляется целевой уровень (`B1` или `B2`),
- вызывается внешний LLM API,
- возвращается адаптированный текст.

### 2. Хранение истории
Каждый успешный результат сохраняется в `TranslationHistoryService`:
- исходный текст,
- адаптированный перевод,
- целевой уровень.

История хранится **на уровне пользовательской сессии**.

### 3. Проверка качества адаптации
При скачивании отчета приложение:
- берет все сохраненные адаптированные тексты,
- отправляет каждый текст в **Текстометр API**,
- получает:
  - `level_number`
  - `level_comment`
- сравнивает результат с целевым диапазоном уровня.

### 4. Excel-отчет
Отчет содержит:
- лист `Results` — таблица по каждому адаптированному тексту,
- лист `Scale` — шкала диапазонов уровней,
- лист `Chart` — диаграмма с процентным соотношением:
  - адаптирован на назначенный уровень,
  - не адаптирован на назначенный уровень.

## Описание основных классов

### `TranslatorController`
Главный MVC-контроллер приложения.

Отвечает за:
- отображение страницы `/translator`,
- обработку формы перевода,
- запуск генерации Excel-отчета,
- очистку истории после выгрузки отчета.

### `TranslatorService`
Сервис адаптации текста.

Здесь:
- хранятся шаблоны промптов,
- подставляется выбранный пользователем уровень,
- выполняется вызов внешнего генеративного API.

### `TranslationHistoryService`
Сессионное хранилище истории.

Сохраняет результаты текущего пользователя:
- исходный текст,
- адаптированный текст,
- целевой уровень.

### `TextometrService`
Клиент для **Текстометр API**.

Использует endpoint анализа текста и возвращает:
- численный уровень сложности,
- текстовый комментарий уровня.

### `ExcelExportService`
Генератор Excel-отчета.

Формирует:
- лист с результатами проверки,
- лист со шкалой уровней,
- диаграмму с процентным распределением успешных/неуспешных адаптаций.

### `LevelRanges`
Вспомогательный класс с диапазонами уровней.

Используется для проверки:
- попал ли адаптированный текст в требуемый диапазон,
- соответствует ли результат назначенному уровню.

## Основной сценарий работы

1. Пользователь открывает страницу `/translator`
2. Вводит текст
3. Выбирает:
   - промпт,
   - уровень (`B1` или `B2`)
4. Нажимает **Перевести**
5. Результат показывается на странице и сохраняется в историю
6. После накопления нескольких примеров пользователь нажимает **Скачать Excel**
7. Приложение:
   - анализирует все переводы через Текстометр,
   - строит отчет,
   - отдает `.xlsx`,
   - очищает историю сессии

## Что важно знать

### История хранится в сессии
Это значит:
- у каждого пользователя своя история,
- после скачивания отчета история очищается,
- при перезапуске приложения данные не сохраняются.

### Проверяется именно результат адаптации
В Excel анализируются **адаптированные тексты**, а не исходные.
Это важно, потому что задача приложения — проверить, насколько хорошо модель сумела привести текст к нужному уровню.

### Диаграмма строится не средствами встроенного Excel chart
Для более стабильного визуального результата график генерируется как изображение и вставляется в Excel-файл.

## Что можно улучшить дальше

Ниже идеи для развития проекта:

- хранить историю в БД, а не только в сессии;
- добавить авторизацию пользователей;
- сохранять отчеты в файловое хранилище;
- поддержать больше уровней сложности;
- сделать сравнение нескольких промптов на одном и том же тексте;
- добавить экспорт не только в Excel, но и в PDF;
- показать историю адаптаций прямо на веб-странице.

## Идея проекта

По сути это маленький стенд для проверки качества адаптации текста.

Он решает сразу две задачи:
1. **генерирует упрощенный текст**,  
2. **проверяет, действительно ли он стал проще до нужного уровня**.

В этом и есть главная ценность проекта: не просто получить ответ от модели, а уметь его **проверить количественно**.
